# 市场分析自动化工作流

> 本skill是QuantNova项目的专属工作流，依托项目生存。

## 触发条件

当用户输入包含以下关键词时自动激活：
- "分析全部市场" / "分析所有品种"
- "分析XX品种" / "分析XX"
- "扫描市场" / "扫描品种"
- "全市场分析"
- "期货分析" / "证券分析"

---

## 工作流一：日常扫描（主流程）

### Step 1: 数据同步
```
获取全品种K线数据（250日）
    ↓
获取技术指标（通达信MCP优先/本地计算）
    ↓
获取基本面数据（可选）
    ↓
数据时效性检查
    ↓
保存到DuckDB
```

### Step 2: 全品种扫描
```
遍历所有品种
    ↓
计算技术指标
    ↓
计算趋势/动量/波动率因子
    ↓
生成初步信号
    ↓
筛选活跃品种（排除僵尸品种）
```

### Step 3: 信号筛选（自优化）

```
1. 加载当前最优参数
   - 从参数配置文件读取阈值
   - 默认: EMA交叉 + RSI阈值 + MACD确认
   ↓
2. 应用当前参数生成信号
   - 多头: EMA20 > EMA60 且 RSI > {多头RSI阈值}
   - 空头: EMA20 < EMA60 且 RSI < {空头RSI阈值}
   ↓
3. 信号强度评分（-100 到 +100）
   - 多头: 正分
   - 空头: 负分
   - 震荡: 接近0
   ↓
4. 筛选出有明确信号的品种
   ↓
5. 记录本次信号生成参数（用于回测）
```

**自优化参数说明**：
- `多头RSI阈值`: 默认55，可通过回测优化
- `空头RSI阈值`: 默认45，可通过回测优化
- `EMA周期`: 默认20/60，可通过回测优化
- `MACD确认`: 是否要求MACD柱状图确认

### Step 4: Context组装
```
技术面Context:
    - K线形态
    - 均线排列
    - 动量指标状态
    - 波动率状态
    ↓
基本面Context（可选）:
    - 库存/仓单（期货）
    - 财务数据（证券）
    - 行业数据
    ↓
市场环境Context:
    - 大盘趋势
    - 板块轮动
    - 资金流向
```

### Step 5: 经验检索
```
从记忆系统检索:
    - 历史相似行情
    - 过去交易记录
    - 因子表现记录
    ↓
相关经验排序
    ↓
提取关键经验点
```

### Step 6: LLM推理
```
组装Prompt:
    - 市场数据
    - 技术指标
    - 基本面信息
    - 历史经验
    ↓
调用LLM推理引擎
    ↓
生成初步分析结论
```

### Step 7: 鹰鸽辩论
```
多头观点（鹰派）:
    - 看涨理由
    - 置信度
    ↓
空头观点（鸽派）:
    - 看跌理由
    - 置信度
    ↓
辩论交锋:
    - 互相质疑
    - 补充论据
    ↓
最终结论:
    - 综合判断
    - 概率区间
```

### Step 8: 决策简报
```
输出格式:
    - 市场评估
    - 操作建议
    - 风险提示
    - 置信度
```

## 工作流二：因子进化（闭环）

### Step 1: 种子因子
```
初始因子库:
    - 趋势因子（EMA/MA）
    - 动量因子（RSI/MACD/KDJ）
    - 波动率因子（ATR/BOLL）
    - 成交量因子（OBV/VWAP）
    ↓
种子因子选择
```

### Step 2: 信号生成参数优化（新增）
```
参数空间定义:
    - RSI阈值: [40, 45, 50, 55, 60]
    - EMA周期: [(10,30), (20,60), (30,90)]
    - MACD确认: [是, 否]
    ↓
网格搜索最优参数:
    - 回测历史数据
    - 计算每个参数组合的胜率/盈亏比
    ↓
输出最优参数:
    - 最优RSI阈值
    - 最优EMA周期
    - 最优MACD设置
    ↓
保存到参数配置文件
```

### Step 3: 候选生成
```
因子变异:
    - 参数调整
    - 组合优化
    - 新因子生成
    ↓
候选因子池
```

### Step 4: 代码验证
```
语法检查
    ↓
逻辑验证
    ↓
边界条件测试
    ↓
代码质量评分
```

### Step 5: 执行计算
```
回测历史数据
    ↓
计算因子值
    ↓
生成信号序列（使用Step 2优化后的参数）
    ↓
计算收益序列
```

### Step 6: IC/ICIR评估
```
IC（信息系数）:
    - 因子值与未来收益的相关性
    - |IC| > 0.03 为有效因子
    ↓
ICIR（信息比率）:
    - IC均值 / IC标准差
    - |ICIR| > 0.5 为稳定因子
    ↓
因子有效性评分
```

### Step 6: Walk-Forward验证
```
滚动窗口回测:
    - 训练期: 120日
    - 测试期: 20日
    ↓
样本外表现评估
    ↓
过拟合检测
```

### Step 7: 门控决策
```
通过条件:
    - IC > 0.03
    - ICIR > 0.5
    - 回测胜率 > 55%
    - 最大回撤 < 20%
    ↓
通过: 进入因子库
    ↓
未通过: 丢弃或返回Step 2
```

### Step 8: 经验记忆
```
记录:
    - 因子参数
    - 表现指标
    - 市场环境
    ↓
存储到记忆系统
```

### Step 9: 反馈优化
```
分析因子表现:
    - 哪些市场环境有效
    - 哪些市场环境失效
    ↓
调整因子权重
    ↓
优化因子组合
```

## 工作流三：自优化闭环

### Step 1: 交易执行
```
根据信号执行:
    - 期货: TqSdk下单
    - 证券: 模拟记录
    ↓
记录执行细节:
    - 价格
    - 数量
    - 时间
```

### Step 2: 结果记录
```
记录交易结果:
    - 盈亏
    - 持仓时间
    - 信号准确率
    ↓
保存到数据库
```

### Step 3: 轨迹分析
```
分析交易轨迹:
    - 入场时机
    - 出场时机
    - 仓位管理
    ↓
识别成功/失败模式
```

### Step 4: 故障归因
```
失败交易分析:
    - 信号错误
    - 时机错误
    - 仓位错误
    - 市场环境变化
    ↓
归因到具体因子
```

### Step 5: LLM反思
```
调用LLM分析:
    - 交易决策过程
    - 错误原因
    - 改进方向
    ↓
生成反思报告
```

### Step 6: 规则优化
```
基于反思结果:
    - 调整信号阈值
    - 优化仓位计算
    - 改进止损止盈
    ↓
生成新规则
```

### Step 7: 过拟合审计
```
检测规则是否过拟合:
    - 样本内vs样本外表现
    - 参数敏感性
    - 市场环境适应性
    ↓
通过: 进入规则库
    ↓
未通过: 返回Step 5
```

### Step 8: 规则晋升
```
规则成熟度评估:
    - 临时规则 → 候选规则 → 正式规则
    ↓
晋升条件:
    - 连续20次有效
    - 覆盖多种市场环境
    ↓
更新规则库
```

### Step 9: 经验存储
```
存储:
    - 规则变更记录
    - 表现对比
    - 市场环境标签
    ↓
更新记忆系统
```

### Step 10: 闭环
```
返回Step 1（交易执行）
    ↓
持续迭代优化
```

## 容错机制

1. **数据获取失败**: 跳过该品种，继续处理其他品种
2. **计算异常**: 使用默认值，标记为"数据异常"
3. **超时处理**: 单品种超时30秒自动跳过
4. **部分完成**: 即使部分品种失败，也要输出已分析品种的结果

---

## 注意事项

1. **不自动下单**: 只提供分析建议，不执行交易（除非用户明确授权）
2. **数据第一**: 所有分析基于真实数据，禁止模拟数据
3. **风险提示**: 每次分析必须包含风险提示
4. **时效性**: 明确定义数据截止时间
