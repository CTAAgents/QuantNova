> 版本：v2.1.0 | 创建日期：2026-06-20 | 简化版
> 状态：核心闭环已验证，已简化架构

---

## 1. 系统定位

QuantNova 是一个**推理重于规则的量化交易决策辅助系统**，支持期货和证券双市场。

- **核心理念**：不依赖硬编码规则，而是让推理层根据市场状态动态生成交易规则。
- **核心机制**：通过 AI 辩论机制对抗交易人性弱点。
- **适用市场**：中国期货 + 证券市场。

## 4. 核心工作流程

```
自动扫描（每日）
    ↓
五维初筛：技术指标、资金流向、基本面、期限结构、情绪指标打分
    ↓
Reasoner 推理：生成初步多空判断
    ↓
🦅🕊️ AI 辩论（强制关卡）
    ├── 鹰派生成做多论据
    ├── 鸽派生成做空论据
    └── 计算分歧度与风险调整因子
    ↓
蒙特卡洛模拟：重排 10,000 次，评估破产概率与最坏情景
    ↓
风控检查：熔断阈值、过拟合检测
    ↓
输出建议：仅向用户提交通过辩论和风控的信号
```

---

## 5. 核心原则（铁律）

| 原则 | 说明 |
|------|------|
| **数据第一** | 绝对禁止使用模拟数据，必须用真实数据 |
| **系统分析** | 必须走完整流程，禁止简化逻辑 |
| **推理重于规则** | 规则动态生成，非硬编码 |
| **辩论验证** | 未经辩论的建议不得提交给用户 |
| **自优化** | 持续记录、反思、进化 |

## 6. 优缺点总结

### 优点

| 维度 | 评价 |
|------|------|
| 辩论纠偏 | 有效对抗人性弱点（FOMO、确认偏误） |
| 风控完善 | Davey 框架 + 蒙特卡洛 + 熔断，实盘生存率高 |
| 可解释性 | 辩论留痕，逻辑白盒化 |
| 前瞻设计 | Multi-Agent 架构，符合 Quant 4.0 趋势 |

### 缺点

| 维度 | 评价 |
|------|------|
| LLM 幻觉 | 论据可能基于错误前提或编造数据 |
| 架构臃肿 | 9 层 + 3 框架，维护成本高，模块耦合难 |
| RL 落地难 | 强化学习在期货实盘中成功率极低 |
| 数据治理缺失 | 未明确数据对齐与异常处理机制 |

---

## 7. 最佳实践建议

1. **取其精华**：借鉴"辩论纠偏"思维和"Davey 风控"框架。
2. **弃其糟粕**：暂时搁置 RL 模块和过于复杂的 NLP 交互。
3. **简化架构**：先跑通"扫描→辩论→风控"核心闭环，再叠加模块。
4. **数据核实**：永远不要盲信 LLM 引用的数据，务必去原始源（Wind/钢联）核对。
5. **小资金试水**：新策略先通过小资金实盘孵化，验证后再加仓。

---

## 8. 一句话总结

> **QuantNova 是一套"理念超前、架构复杂、落地尚早"的量化系统。它最大的价值在于用 AI 辩论机制对抗交易人性弱点，而非提供高收益预测。把它当作一个"偏见过滤器"和"风控助手"，而非"全能分析师"，才能发挥其真正价值。**
