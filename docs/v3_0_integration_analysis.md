## 二、核心底线原则实现

V3.0方案的6条底线原则：

| 原则 | QuantNova 实现方式 |
|------|-------------------|
| 仅承担逻辑推演 | `ReasoningEngine` 输出分析报告，不执行交易 |
| 不编造数据 | `HallucinationDetector` + 数据校验 |
| 不输出确定性结论 | `DebateEngine` 提供多视角，输出概率区间 |
| 不替代人工风控 | 系统仅提供决策简报，最终决策权在人 |

## 三、数据层增强

### 3.1 数据可信度分级

```
1级（最高）：交易所、统计局官方数据
2级：头部券商研报、产业调研
3级：Wind、Bloomberg终端数据
4级（最低）：自媒体、散户舆情
```

### 3.2 冲突裁决规则

```python
# 优先级规则
1. 最新数据优先
2. 高可信度数据源优先
3. 统计口径匹配数据优先
```

### 3.3 异常值处理

```python
# 三级异常分类
制度性异常 → 标记，权重降低50%
短期脉冲异常 → 标记，权重降低30%
长期趋势断裂 → 重点分析，作为情景推演输入
```

## 四、提示词工程增强

### 4.1 三级自适应Prompt模板

| 模板类型 | 适用场景 | 推理步骤 |
|----------|----------|----------|
| 极简模板 | 纯指标解读 | 3步精简推理 |
| 标准模板 | 常规日线研判 | 7步对抗链式推理 |
| 深度模板 | 重仓研判、跨品种对冲 | 完整约束+专项校验 |

### 4.2 幻觉硬约束前置指令

1. 严禁编造未提供的数据
2. 关键数据缺失直接说明
3. 严格分块输出事实与观点
4. 禁止100%确定性结论
5. 引用低可信度数据必须标注风险

## 五、推理架构增强

### 5.1 模块职责拆分

```
量化计算引擎 → 全权负责数值运算
RAG检索模块 → 召回事实素材
主推理大模型 → 逻辑推演、情景拆分
校验模块 → 分层拦截错误
```

### 5.2 故障降级机制

| 故障类型 | 一级降级 | 二级降级 |
|----------|----------|----------|
| 量化工具宕机 | 标注工具离线 | 简化版指标计算 |
| RAG检索故障 | 标注历史参考缺失 | 人工介入补充 |

## 六、输出校验增强

### 6.1 机器校验规则

- 数值一致性校验（偏差>2%驳回）
- 金融逻辑规则引擎（5类强制重推场景）

### 6.2 人工复核评分卡

| 核验项 | 5分标准 | 3分标准 | 0分标准 |
|--------|---------|---------|---------|
| 数据源溯源 | 全部标注来源等级 | 部分标注 | 未标注 |
| 多空逻辑对称 | 双方理由各≥3条 | 有一方偏弱 | 只有单边逻辑 |
| 流动性风险提示 | 明确标注并分析 | 仅提及未分析 | 完全忽略 |
| 极端情景完整性 | 三种情景详细推演 | 缺一种情景 | 只有单种情景 |
| 无确定性结论 | 所有预判带概率区间 | 部分带概率 | 出现100%确定表述 |

| 指标 | 定义 | 准入门槛 |
|------|------|----------|

| 风险情景完整度 | 三种情景均被覆盖的比例 | ≥80% |
